# Connect Wallet

Use <code>wallet_connect</code> when you want a single request that both asks the user to connect and returns the account data your app usually needs right away.

On success, the method grants read permissions for the selected account and returns the account id, addresses, wallet type, and active network state.

<Callout type="info">
  If the user has already granted permissions, no connection request popup will
  appear, ensuring a smooth user experience.
</Callout>

## After connecting

- Use [wallet_getAccount](/wallet/get-account) when you need the current connected account again later.
- Use [wallet permissions](/wallet/permissions) to understand when read access is reused versus re-requested.
- Use [wallet events](/wallet/events) so your UI stays in sync when the user changes accounts, networks, or disconnects.
