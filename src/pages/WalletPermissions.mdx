# Xverse Wallet Permissions

Permissions let your app read account or wallet state without prompting the user again on every request.

<Callout type="info">
  In <code>sats-connect@4.2.1</code>, <code>wallet_connect</code> is still the
  simplest way to request the initial account-read permission. Use
  <code>wallet_requestPermissions</code> when you want to trigger or retry the
  permission flow explicitly.
</Callout>

## Inspect or revoke permissions

- [wallet_getCurrentPermissions](/wallet/get-current-permissions) shows the permissions currently granted to your app
- [wallet_renouncePermissions](/wallet/renounce-permissions) clears them again

## Signatures still prompt per request

Bitcoin, Stacks, and Spark signing or transaction methods still require explicit user approval each time you call them.
