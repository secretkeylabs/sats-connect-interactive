# signMessage

Use the <code>signMessage</code> method to request the user to sign a message
with their wallet's Bitcoin address. This is commonly used for authentication,
identity verification, or proof-of-ownership.

## Parameters

<ParamTable
  params={[
    {
      name: "address",
      type: "string",
      required: true,
      description: "The Bitcoin address to sign with.",
    },
    {
      name: "message",
      type: "string",
      required: true,
      description: "The message to sign.",
    },
    {
      name: "protocol",
      type: "'ECDSA' | 'BIP322'",
      required: false,
      description:
        "Signing protocol. Defaults to ECDSA. Use BIP322 for taproot addresses.",
    },
  ]}
/>

## Response

On success, <code>signMessage</code> returns the signature and message hash:

<ParamTable
  params={[
    {
      name: "signature",
      type: "string",
      required: true,
      description: "The message signature.",
    },
    {
      name: "messageHash",
      type: "string",
      required: true,
      description: "Hash of the signed message.",
    },
  ]}
/>

<Callout type="tip">
  Use **ECDSA** for legacy and SegWit addresses. Use **BIP322** for taproot
  (P2TR) addresses. The wallet user will see the message content before
  approving the signing request.
</Callout>
